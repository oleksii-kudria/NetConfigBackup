# task-010-mikrotik-export-command-cleanup.md

## Мета
Привести команду отримання конфігурації з MikroTik у відповідність до офіційної документації RouterOS та спростити реалізацію бекапу.

Необхідно **прибрати використання `/export show-sensitive=false`** та залишити **лише стандартну команду `/export`**, оскільки за замовчуванням `/export` **не відображає чутливу інформацію**.

---

## Контекст
У попередніх задачах для отримання конфігурації MikroTik використовувалась команда:

```
/export show-sensitive=false
```

Відповідно до документації MikroTik:
- `/export` **за замовчуванням не включає чутливі дані**
- параметр `show-sensitive` не є обовʼязковим
- використання зайвих параметрів ускладнює код та обробку помилок

---

## Вимоги

### 1) Зміна команди export
У всіх місцях коду, де виконується отримання конфігурації MikroTik:

❌ Прибрати:
```
/export show-sensitive=false
```

✅ Використовувати:
```
/export
```

---

### 2) Оновлення реалізації
Перевірити та оновити наступні файли (якщо використовуються):

- `src/app/mikrotik/client.py`
- `src/app/mikrotik/backup.py`

Вимоги:
- команда `/export` має бути єдиною
- не повинно залишитись fallback-логіки, повʼязаної з `show-sensitive`
- код має бути простішим та однозначним

---

### 3) Вплив на diff та нормалізацію
- Логіка нормалізації та diff (task-009) **не змінюється**
- Очікується, що `/export` без параметрів:
  - дає стабільніший та чистіший output
  - зменшує кількість шуму у diff

---

### 4) Логування
- У логах більше **не повинно зʼявлятись** згадок про `show-sensitive`
- У DEBUG-режимі допустимо логувати:
```
DEBUG | executing mikrotik command='/export'
```

Без додаткових параметрів.

---

## Acceptance criteria
- У коді використовується **тільки** команда `/export`
- Відсутні згадки `show-sensitive` у коді та логах
- Бекап MikroTik продовжує коректно працювати
- Конфіг не містить чутливих даних
- Diff-перевірка працює без змін

---

## Результат
NetConfigBackup використовує просту, документовану та стабільну команду `/export` для зняття конфігурації з MikroTik, без зайвих параметрів і зайвої логіки.
