## 2) Опис реалізованого функціоналу (UA)

У розділі українською мовою необхідно описати, що вже реалізовано:

- запуск через `scripts/run.py`
- підтримка DEBUG-режиму (`--debug`)
- робота з `devices.yml`
- підтримка `config/local.yml` (не зберігається у git)
- логування з керуванням рівнем
- визначення BACKUP_DIR:
  - CLI
  - local.yml
  - fallback `./backup/`
- MikroTik:
  - зняття конфігурації через `/export`
  - збереження конфігів у файлову структуру
  - визначення змін через hash + normalized diff
  - створення binary backup через `/system backup save`
- розділення:
  - текстовий backup (audit)
  - binary backup (disaster recovery)

Опис має бути стислим, але інженерно точним.

## 3) MikroTik: користувач і група (UA)

Додати окремий підрозділ з прикладом створення:

### Вимоги
- імʼя користувача — випадкове (наприклад `bk_user_01`)
- імʼя групи — випадкове (наприклад `bk_group_01`)
- тільки мінімально необхідні права
- жодних небезпечних сервісів (winbox, api, web, etc.)

### Приклад (UA)
```mikrotik
/user group add name=bk_group_01 policy=\
ssh,ftp,read,write,policy,test,sensitive,\
!local,!telnet,!reboot,!winbox,!password,\
!web,!sniff,!api,!romon,!rest-api

/user add name=bk_user_01 group=bk_group_01 password=STRONG_PASSWORD
```

Також коротко пояснити:
- для чого потрібні ці права
- чому не рекомендується використовувати `full`
